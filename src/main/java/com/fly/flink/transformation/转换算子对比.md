在 Flink 的 `DataStream` API 中，转换算子（Transformation）是数据处理的核心，它们将输入的 `DataStream` 转换为新的
`DataStream`，以构建复杂的数据处理拓扑。下表对常用的转换算子进行了分类对比：

| 类别        | 算子名称                                                    | 输入 → 输出类型                                                   | 核心功能                                           | 输入/输出关系 | 关键特征/使用场景                                       | 示例代码                                                                               |
|:----------|:--------------------------------------------------------|:------------------------------------------------------------|:-----------------------------------------------|:--------|:------------------------------------------------|:-----------------------------------------------------------------------------------|
| **基本转换**  | `map`                                                   | `DataStream` → `DataStream`                                 | 对每个元素应用一个函数。                                   | 一对一     | 数据清洗、类型转换、字段提取。                                 | `dataStream.map(s -> s.toUpperCase())`                                             |
|           | `filter`                                                | `DataStream` → `DataStream`                                 | 根据条件保留满足要求的元素。                                 | 一对一     | 数据筛选、剔除无效数据。                                    | `dataStream.filter(s -> s.length() > 5)`                                           |
|           | `flatMap`                                               | `DataStream` → `DataStream`                                 | 将每个元素拆分为0个、1个或多个元素。                            | 一对多     | 文本分词（将句子拆成单词）、扁平化嵌套结构。                          | `dataStream.flatMap((s, out) -> { for (String w: s.split(" ")) out.collect(w); })` |
| **聚合与窗口** | `keyBy`                                                 | `DataStream` → `KeyedStream`                                | 按指定的Key对数据进行逻辑分区，相同Key的数据会进入同一个分区。             | 分组标记    | 为后续的聚合或窗口操作做准备，是状态计算的基石。                        | `dataStream.keyBy(event -> event.getUserId())`                                     |
|           | `reduce`                                                | `KeyedStream` → `DataStream`                                | 基于当前值和上一个聚合结果，进行滚动聚合。                          | 分组内聚合   | 实时累加（如求和、计数），要求函数满足结合律。                         | `keyedStream.reduce((a, b) -> a + b)`                                              |
|           | `sum`/`min`/`max`/`minBy`/`maxBy`                       | `KeyedStream` → `DataStream`                                | 对KeyedStream进行简单的滚动聚合操作。                       | 分组内聚合   | 语法更简洁，适用于求和、求最值等常见操作。                           | `keyedStream.sum("fieldName")`                                                     |
|           | `window`                                                | `KeyedStream` → `WindowedStream`                            | 将流数据切分成有限大小的“桶”进行处理。                           | 窗口定义    | 支持滚动(Tumbling)、滑动(Sliding)、会话(Session)等时间或计数窗口。 | `keyedStream.window(TumblingEventTimeWindows.of(Time.minutes(5)))`                 |
| **多流合并**  | `union`                                                 | `DataStream<T>` → `DataStream<T>`                           | 合并**多个**类型相同的数据流。                              | 多合一     | 简单合并，不进行去重，不保证元素顺序。常用于合并多源同类型日志。                | `stream1.union(stream2, stream3)`                                                  |
|           | `connect`                                               | `DataStream<T>`, `DataStream<R>` → `ConnectedStreams<T, R>` | 连接**两个**可能类型不同的数据流，保留各自的类型。                    | 二合一     | 功能强大，两个流可以共享状态并分别处理。常用于关联不同数据源。                 | `stream1.connect(stream2)`                                                         |
|           | `coMap`/`coFlatMap`                                     | `ConnectedStreams<T, R>` → `DataStream`                     | 对`ConnectedStreams`中的两个流分别应用`map`或`flatMap`函数。 | 分别处理    | 用于处理`connect`后的流，为每个流定义独立的处理逻辑。                 | `connectedStreams.map(new CoMapFunction...(){...})`                                |
| **数据重分区** | `shuffle`                                               | `DataStream` → `DataStream`                                 | 将数据随机均匀地分配到下游算子的并行任务中。                         | 随机分配    | 用于解决数据倾斜，但不保证确定性。                               | `dataStream.shuffle()`                                                             |
|           | `rebalance`                                             | `DataStream` → `DataStream`                                 | 采用Round-Robin轮询算法，将数据均匀地分配到下游任务。               | 轮询分配    | 比`shuffle`更均匀，常用于在分区前进行负载均衡。                    | `dataStream.rebalance()`                                                           |
|           | `rescale`                                               | `DataStream` → `DataStream`                                 | 基于轮询，但仅在算子上下游的任务子集间进行数据分发。                     | 局部轮询    | 减少跨节点网络传输，优化性能，适合当数据不需要全局重新分布时。                 | `dataStream.rescale()`                                                             |
|           | `broadcast`                                             | `DataStream` → `DataStream`                                 | 将输入数据的**一个副本**发送到下游算子的**每一个**并行实例中。            | 一对多广播   | 用于将一些静态或配置数据（如规则）广播到所有并发任务中。                    | `dataStream.broadcast()`                                                           |
|           | `global`                                                | `DataStream` → `DataStream`                                 | 将所有数据都发送到下游算子的第一个并行实例（分区0）进行处理。                | 合并到单个任务 | 会将所有数据汇集到一个分区，可能导致严重的性能瓶颈，应谨慎使用。                | `dataStream.global()`                                                              |
|           | `partitionCustom`                                       | `DataStream` → `DataStream`                                 | 使用用户自定义的`Partitioner`，根据记录的Key决定数据发往哪个下游分区。    | 自定义分配   | 当内置分区策略无法满足特定需求时，可以完全自定义数据分发规则。                 | `dataStream.partitionCustom(myPartitioner, keySelector)`                           |
| **高级操作**  | `iterate`                                               | `DataStream` → `IterativeStream`                            | 在流上创建一个“反馈回路”，将一部分数据反馈回流开头进行迭代处理。              | 流内循环    | 适用于需要反复迭代计算的机器学习或图计算场景。                         | `dataStream.iterate().withFeedbackType(...)`                                       |
|           | `extractTimestamp` & `assignTimestampsAndWatermarks`    | `DataStream` → `DataStream`                                 | 从数据元素中提取事件时间戳，并生成水位线（Watermark）以处理乱序数据。        | 时间提取与标记 | 使用事件时间（Event Time）语义进行窗口计算时，这是必需的步骤。            | `dataStream.assignTimestampsAndWatermarks(...)`                                    |
|           | `async I/O` (AsyncDataStream.unorderedWait/orderedWait) | `DataStream` → `DataStream`                                 | 用于高效地与外部系统（如数据库）进行异步交互，提升系统吞吐量。                | 异步交互    | 在需要访问外部存储来丰富数据流的场景下，是性能优化的关键手段。                 | `AsyncDataStream.unorderedWait(dataStream, asyncFunc, timeout, unit, capacity)`    |
